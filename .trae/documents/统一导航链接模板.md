# 统一导航链接模板

## 问题分析
用户报告修改"按房间找"和"按分类找"链接时，"管理分类"链接没有变化，要求统一成一个模板。

## 原因分析
1. 前两个链接（按房间找、按分类找）使用查询参数（`?filter=room` 和 `?filter=category`），并有 `data-filter` 属性用于处理激活状态
2. 第三个链接（管理分类）使用命名 URL 模式（`{% url 'items:manage_categories' %}`），没有 `data-filter` 属性
3. 激活状态的 JavaScript 代码只处理带有 `data-filter` 属性的链接
4. 管理分类链接可能位于不同的模板中，或者使用不同的样式处理

## 修复方案

### 1. 更新导航链接结构
- 为管理分类链接添加 `data-filter` 属性，用于统一的激活状态处理
- 确保所有三个链接使用相同的 HTML 结构和 CSS 类
- 更新 JavaScript 代码，确保所有链接都能正确处理激活状态

### 2. 确保模板统一
- 检查管理分类页面是否使用相同的基础模板
- 确保所有导航链接的样式一致
- 统一激活状态的处理逻辑

### 3. 优化用户体验
- 确保导航链接的视觉一致性
- 保持激活状态的清晰反馈
- 确保所有链接都能正确触发页面更新

## 实现步骤

1. **修改导航链接模板**：
   - 为管理分类链接添加 `data-filter` 属性
   - 确保所有链接使用相同的 HTML 结构

2. **更新 JavaScript 代码**：
   - 修改激活状态处理逻辑，确保所有链接都能正确处理
   - 确保管理分类链接也能触发相应的激活状态

3. **测试修复效果**：
   - 检查所有导航链接的激活状态是否正确
   - 验证管理分类链接与其他链接的样式是否一致
   - 确保所有链接都能正常工作

## 预期效果
- 所有三个导航链接使用相同的模板和样式
- 管理分类链接与其他链接的激活状态处理一致
- 统一的视觉设计和用户体验
- 不影响其他功能的正常运行

## 技术要点
- 使用相同的 HTML 结构和 CSS 类
- 统一的 JavaScript 激活状态处理
- 确保模板的一致性
- 保持代码简洁和可维护性
