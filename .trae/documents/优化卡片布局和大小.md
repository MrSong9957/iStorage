# 优化卡片布局和大小计划

## 目标
- 将物品卡片在一排中合理布局，尽可能多的显示更多物品
- 减小卡片大小，同时保持苹果设计风格和极简主义设计
- 只修改必要的代码，不影响其他功能

## 具体修改内容

### 1. 调整物品网格布局
- 修改 `.items-grid` 类的 `grid-template-columns` 属性，将最小宽度从 320px 减小到 180px，允许更多卡片在一排显示
- 减小网格间距从 `var(--spacing-md)` 到 `var(--spacing-sm)`，增加空间利用率

### 2. 减小卡片大小
- 修改 `.item-card` 类的 `padding` 属性，从 `var(--spacing-md)` 减小到 `var(--spacing-sm)`
- 保持苹果设计风格的圆角和阴影效果

### 3. 优化卡片内部结构
- 修改 `.item-card-header` 类的 `margin-bottom` 和 `padding-bottom` 属性，减小内部间距
- 调整 `.item-name` 类的字体大小，从 20px 减小到 16px
- 调整 `.meta-item` 类的间距和字体大小，保持信息清晰的同时减小空间占用

### 4. 确保响应式设计
- 检查并调整响应式布局，确保在不同屏幕尺寸下都能良好显示
- 保持苹果设计风格的一致性和极简主义原则

## 预期效果
- 物品卡片在一排中显示更多，提高空间利用率
- 卡片大小减小，同时保持清晰的信息展示和良好的视觉效果
- 符合苹果设计风格和极简主义设计原则
- 不影响其他功能，确保现有数据的兼容性

## 实现步骤
1. 修改 `static/css/custom.css` 文件中的相关样式类
2. 测试修改后的效果，确保布局合理且美观
3. 验证其他功能不受影响

## 技术要点
- 使用 CSS Grid 实现灵活的卡片布局
- 利用 CSS 变量保持设计一致性
- 遵循苹果设计风格的圆角、阴影和间距原则
- 优先考虑可读性和用户体验

## 文件修改
- 仅修改 `static/css/custom.css` 文件，不涉及其他文件的创建或删除
- 不修改 HTML 结构，仅通过 CSS 调整布局和样式
