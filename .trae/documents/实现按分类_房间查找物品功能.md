# 实现按分类/房间查找物品功能计划

## 目标
- 为导航链接添加分类标签功能，点击后显示按照物品分类或房间排列的页面
- 根据点击的标签动态更新统计卡片的名称和数量
- 符合苹果设计风格和极简主义设计
- 只修改必要的代码，不影响其他功能

## 具体实现内容

### 1. 修改视图函数
- 更新 `find_items` 视图函数，添加分类和房间的统计数据
- 实现根据筛选条件（分类或房间）返回不同的物品列表
- 添加处理筛选参数的逻辑

### 2. 更新 URL 配置
- 确保 `find_items` 路由能够处理不同的筛选参数

### 3. 修改模板文件
- 更新导航链接，添加 URL 参数或使用 AJAX 切换筛选条件
- 动态更新统计卡片的标签和数量
- 动态更新物品列表

### 4. 添加 JavaScript 交互
- 实现导航链接的点击事件处理
- 实现页面内容的动态更新
- 保持苹果设计风格的一致性

### 5. 优化样式
- 确保导航链接的激活状态能够正确显示
- 保持极简主义设计风格

## 预期效果
- 点击「按房间找」链接，页面显示按照房间排列的物品，统计卡片显示房间数
- 点击「按分类找」链接，页面显示按照分类排列的物品，统计卡片显示分类数
- 符合苹果设计风格和极简主义设计原则
- 不影响其他功能，确保现有数据的兼容性

## 实现步骤
1. 修改 `apps/items/views.py` 文件，更新 `find_items` 视图函数
2. 修改 `templates/items/find_items.html` 文件，添加交互逻辑
3. 更新 `static/css/custom.css` 文件，优化样式
4. 测试修改后的效果，确保功能正常
5. 验证其他功能不受影响

## 技术要点
- 使用 Django 视图处理不同的筛选条件
- 使用 JavaScript 实现页面内容的动态更新
- 利用 CSS 实现导航链接的激活状态
- 遵循苹果设计风格和极简主义设计原则

## 文件修改
- `apps/items/views.py`：更新 `find_items` 视图函数
