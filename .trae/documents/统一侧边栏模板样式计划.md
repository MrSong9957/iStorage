# 统一侧边栏模板样式计划

## 问题分析

根据用户提供的截图，当前系统存在以下问题：
1. 导航栏样式不一致：不同页面的导航栏样式不同
2. 标签栏样式不一致：不同页面的标签栏样式不同
3. 模板结构不一致：不同页面使用了不同的模板结构

用户需求：统一模板，一律按照管理分类页面的样式来实现。

## 实施计划

### 1. 统一导航栏样式

**目标**：将所有页面的导航栏样式统一为管理分类页面的样式

**实施步骤**：
- 修改`sidebar.html`模板，调整导航栏的HTML结构和CSS类名
- 确保导航栏的样式与管理分类页面一致
- 调整导航项的激活状态样式

### 2. 统一标签栏样式

**目标**：将所有页面的标签栏样式统一为管理分类页面的样式

**实施步骤**：
- 修改`find_items.html`中的分类标签部分，调整HTML结构和CSS类名
- 确保标签栏的样式与管理分类页面一致
- 调整标签项的激活状态样式

### 3. 统一模板结构

**目标**：确保所有页面使用相同的模板结构

**实施步骤**：
- 分析管理分类页面的模板结构
- 调整`find_items.html`的模板结构，使其与管理分类页面一致
- 确保页面布局、间距、颜色等样式统一

### 4. 调整CSS样式

**目标**：确保所有页面的CSS样式统一

**实施步骤**：
- 分析`custom.css`中的样式定义
- 调整相关CSS类的样式，确保统一
- 移除不必要的CSS样式，避免样式冲突

### 5. 测试与验证

**目标**：验证统一模板后的效果

**实施步骤**：
- 访问所有页面，检查导航栏和标签栏样式是否一致
- 验证页面交互功能是否正常
- 确保页面响应式设计正常

## 预期效果

- 所有页面的导航栏样式统一为管理分类页面的样式
- 所有页面的标签栏样式统一为管理分类页面的样式
- 所有页面使用相同的模板结构
- 页面布局、间距、颜色等样式统一
- 保持原有的交互功能不变

## 实施步骤

1. 修改`sidebar.html`模板，统一导航栏样式
2. 修改`find_items.html`模板，调整分类标签部分
3. 调整CSS样式，确保统一
4. 测试所有页面，验证效果
5. 优化调整，确保完全统一

## 技术要点

- 使用Django模板继承和包含机制，实现代码复用
- 统一CSS类名，避免样式冲突
- 确保响应式设计，适配不同屏幕尺寸
- 保持原有的交互功能不变
- 确保代码的可维护性和扩展性
