# 优化页面交互和模板继承

## 1. 问题分析

### 1.1 分类物品卡片交互效果
- 当前分类物品卡片没有交互效果，而最近物品卡片已经有了点击显示详细信息的功能
- 需要统一所有卡片的交互效果，确保用户体验一致

### 1.2 缺少返回按钮
- 查找物品页面缺少返回按钮，用户需要通过浏览器返回，体验不佳
- 需要在标题左侧添加返回按钮，链接到首页

### 1.3 首页未套用父模板
- 首页 `index.html` 没有继承 `base_template.html`，导致代码重复
- 需要修改首页，使其继承父模板，保持代码一致性

## 2. 修复方案

### 2.1 统一卡片交互效果
- 确保分类物品卡片也有与最近物品卡片相同的 `item-card` 类和数据属性
- 这样它们就能自动使用现有的模态窗口功能

### 2.2 添加返回按钮
- 在 `find_items.html` 中的标题左侧添加返回按钮
- 使用 SVG 图标，保持与整体设计风格一致
- 链接到首页

### 2.3 修改首页继承父模板
- 修改 `index.html`，使其继承 `base_template.html`
- 将现有内容放入 `content` 块中
- 移除重复的 HTML 结构，如导航栏和样式定义

## 3. 实现步骤

### 3.1 修改 find_items.html
1. 在标题左侧添加返回按钮
2. 确保分类物品卡片有正确的类和数据属性

### 3.2 修改 index.html
1. 添加模板继承语句
2. 将内容放入 `content` 块中
3. 移除重复的 HTML 结构

## 4. 技术要点
- 使用 Django 模板继承机制，减少代码重复
- 保持现有 JavaScript 代码不变，通过添加正确的类和数据属性实现功能复用
- 使用 Tailwind CSS 实现样式，保持与整体设计风格一致

## 5. 文件修改清单
- `templates/items/find_items.html`：添加返回按钮，统一分类物品卡片交互效果
- `templates/index.html`：修改为继承 base_template.html
