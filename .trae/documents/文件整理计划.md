# 文件整理计划

## 1. 文件评估与整理

### 根目录文件评估

* `query_db.py`：临时数据库查询脚本，用于查看模型数据，与项目核心功能无关

* `simple_query.py`：临时脚本，用于打印数据库表名，与项目核心功能无关

* `view_db_data.py`：临时脚本，用于直接查询SQLite数据库，与项目核心功能无关

### scripts文件夹文件评估

* `associate_items.py`：一次性脚本，用于关联物品到储物格，已完成其功能

* `check_items.py`：调试脚本，用于检查物品情况，与项目核心功能无关

* `check_qr_codes.py`：调试脚本，用于检查二维码数据，与项目核心功能无关

* `check_users.py`：调试脚本，用于检查用户数据，与项目核心功能无关

* `encrypt_password.py`：一次性脚本，用于加密用户密码，已完成其功能

* `generate_missing_codes.py`：一次性脚本，用于生成物品编号，已完成其功能

* `generate_qr_codes.py`：一次性脚本，用于生成二维码，已完成其功能

* `get_supabase_service_key.py`：辅助脚本，用于获取Supabase服务密钥，可能偶尔有用

## 2. 整理方案

* **删除根目录下的临时查询文件**：`query_db.py`、`simple_query.py`、`view_db_data.py`

* **保留scripts文件夹**，但删除其中的一次性和调试脚本：

  * 删除：`associate_items.py`、`check_items.py`、`check_qr_codes.py`、`check_users.py`、`encrypt_password.py`、`generate_missing_codes.py`、`generate_qr_codes.py`

  * 保留：`get_supabase_service_key.py`（可能偶尔有用）

## 3. 项目结构文档更新

* 在 `project_structure.md` 中记录文件删除情况和剩余文件的用途

## 4. 执行步骤

1. 无需创建项目结构文档，它就在docs文件夹中`project_structure.md`
2. 删除根目录下的临时查询文件
3. 删除scripts文件夹中的不必要脚本
4. 更新 `project_structure.md` 文档
5. 验证项目运行正常，确保现有数据兼容性

## 5. 预期结果

* 项目根目录保持简洁，只包含必要的配置文件和管理脚本

* scripts文件夹只保留有用的辅助脚本

* 项目结构文档清晰记录项目结构和文件用途

* 项目运行正常，现有数据不受影响
