# 优化物品查找页面布局

## 问题分析
用户希望将 `h4` 分类标题（如"卧室"）设置为可点击的子标签，并优化统计卡片和添加按钮的布局，使其符合主流用户习惯。

## 设计方案

### 1. 将 h4 分类标题改为可点击链接
- 将 `<h4 class="group-title">` 改为 `<a href="#" class="group-title-link">`
- 保留原有的视觉样式，同时添加悬停效果和交互反馈
- 实现点击分类标题时可以展开/折叠该分类下的物品列表

### 2. 优化统计卡片和添加按钮布局
- 将三个统计卡片（物品总数、房间数/分类数、添加按钮）调整为更紧凑的网格布局
- 在桌面端保持水平排列，在移动端自动调整为垂直排列
- 添加卡片间的间距，提高视觉层次感

### 3. 增强分类标题的视觉设计
- 添加分类标题的图标或前缀，增强视觉识别度
- 优化分类标题与物品数量的布局关系
- 添加分类标题的交互状态（默认、悬停、点击）

## 实现步骤

1. **修改模板文件**：
   - 将 `find_items.html` 中的 `h4.group-title` 改为 `a.group-title-link`
   - 添加分类标题的点击事件处理

2. **更新 CSS 样式**：
   - 添加 `group-title-link` 类的样式
   - 优化统计卡片的布局
   - 增强分类标题的视觉设计

3. **添加 JavaScript 交互**：
   - 实现分类标题的点击展开/折叠功能
   - 添加平滑过渡效果

## 预期效果
- 分类标题变为可点击的子标签，方便用户快速导航
- 统计卡片和添加按钮布局更加紧凑和直观
- 整体设计符合苹果风格和现代简约主义
- 增强用户交互体验，提高页面的易用性

## 技术要点
- 使用 HTML 语义化标签，提高可访问性
- 采用 CSS Grid 和 Flexbox 实现响应式布局
- 使用 JavaScript 实现平滑的交互效果
- 保持与现有设计风格的一致性
- 优化移动端适配

该方案将使页面更加符合主流用户习惯，提高用户体验和操作效率。
